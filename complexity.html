Algorithm Complexity

Types of complexity

  • Time complexity

    • The amount of time required to finish an algorithm's execution is known as its time complexity. The large O notation is used to express an algorithm's time complexity. The asymptotic notation used to describe the temporal complexity is big O notation. The time complexity is mostly determined by counting the number of steps required to complete the task.

  • Space complexity

    • The quantity of space required to solve a problem and create an output is the space complexity of an algorithm. Spacecomplexity is written in big O notation in the same way that time complexity is.

Asymptotic Notation

Data structure is a method of efficiently arranging data, and efficiency may be evaluated in terms of time or space. As a result, the optimal data structure is one that takes the least amount of time and memory space to complete all of its operations. We'd focus on time complexity rather than space complexity, and by determining time complexity,we'd be able to determine which data structure is optimal for an algorithm.

The following are some of the most used asymptotic notations for calculating an algorithm's running time complexity.

  1. Big Ο Notation
    2.

    The upper bound of an algorithm's execution time is expressed using the notation O(n). It calculates the worst-case time complexity, or the time it will take an algorithm to finish.

    For example, for a function f(n)
    Ο(f(n)) = { g(n) : there exists c > 0 and n0 such that f(n) ≤ c.g(n) for all n > n0. }
